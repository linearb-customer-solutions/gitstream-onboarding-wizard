5. Designate a gitStream User Account

gitStream automation rules are executed by the user account configured when you authorize LinearB. A dedicated GitLab user such as gitstream-cm is recommended for clarity and auditing.

Create a Gitlab user containing gitstream-cm in the username. Invite this user to your group or projects and give it at least the Maintainer role so it can run pipelines. Tip: you can create a user with the same email address as your own, e.g., user+gitstream-cm@organization.com.

NOTE. If you can't create a dedicated user, as a workaround you can use an admin account, adjust the username to include gitstream-cm here. This will cause gitStream to ignore comments from your acccount to prevent getting in an infinite loop.

Connect to LinearB making sure you're logged in as gitstream-cm on GitLab and as an admin on LinearB.
